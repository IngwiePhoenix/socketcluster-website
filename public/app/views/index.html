What is SocketCluster?

SocketCluster is an open source, multi-process environment for realtime apps and the Internet of Things (IoT). It's a free (no lock-in), self-hosted alternative to Backend as a Service solutions like Firebase, Pusher and PubNub (for those who need more control over their realtime data and backend processes).

SC is not a full-stack framework - It only handles process-level architecture (clustering, resilience, error logging) and lets you take care of the rest - It works well with Express and other modular Node.js frameworks.

SC can scale linearly (on a single host - tested with up to 16 cores) and horizontally (across multiple hosts).

Pub/sub channels can be synced across multiple instances by hooking them into distributed message queues (AMQP, MQTT) - SC store processes provide all the hooks/events necessary.

Technical Features

  • Scales linearly as you add more CPU cores and workers (assuming a decent OS) - Tested with up to 16 cores.
  • A client can publish events to all other interested clients (client-to-client pub/sub)
  • Client sockets automatically reconnect if they lose the connection.
  • SocketCluster processes automatically respawn on crash - Worker and load balancer crashes are invisible to users.
  • Comes with an in-memory session data store.
  • Sockets which are bound to the same browser (for example, across multiple tabs) share the same session.
  • Emit event notifications on any session to notify all sockets that belong to it.
  • SC was optimized to work with single page apps and can run on IoT devices which support JavaScript.
